# 🏊 Public Swimming Pool Finder - Skill.md

## 1. 📌 프로젝트 개요

이 프로젝트는 사용자가 한국의 특정 지역을 검색하면
해당 지역 또는 주변의 공공 수영장을 찾을 수 있는 웹 사이트를 만드는 것을 목표로 한다.

---

## 2. 🎯 핵심 기능

### 2.1 지역 검색 기능

* 사용자가 지역명(예: 서울, 부산 등)을 입력
* 해당 지역의 공공 수영장 목록을 필터링하여 표시
* 시도(서울, 경기...), 시군구(강남구, 수원시...), 시설명(잠실수영장...) 모두 검색 가능


### 2.2 수영장 정보 제공 (Google 정보)
* 주소
* 실내/야외, 25m or 50m, 공공시설
* 기타 정보 (선택: 운영시간, 요금 등)
* 카드 클릭 시 상세 정보 확인 (운영시간, 레인 수, 수심, 전화번호, 공식 사이트)


### 2.3 지도 표시 기능 (Naver지도)

* 검색 이후에 클릭시 지도 표시
* 지도는 **Naver Map API** 사용
* 검색된 수영장 위치를 마커로 표시

---

## 3. 🗂 데이터 구조

### 3.1 데이터 저장 방식

* `/data/Data.json` 파일에 공공 수영장 정보 저장
* 별도의 데이터베이스 없이 JSON 기반으로 관리

### 3.2 Data.json 예시

```json
[
  {
    "name": "종로 올림픽기념국민생활관",
    "region": "서울",
    "address":"종로구",
  },
]
```

---

## 4. 🔄 동작 흐름 (Flow)

1. 사용자가 지역 입력
2. Data.json에서 해당 지역 데이터 필터링
3. 결과 검색 토대로 수영장정보는 name의 결과검색을 토대로 Google 정보를 가져오고  
4. name의 결과검색을 토대로 수영장 정보는 Naver Map API 으로 수영장 위치 마커 표시

---

## 5. 🧠 개발 규칙 (Rules)

### 5.1 데이터 관련

* 모든 수영장 데이터는 Data.json에서만 관리
* 컴포넌트 내부에 하드코딩 금지
* 코드를 변경할 시에 반드시 물어볼 것 

### 5.2 지도 관련

* 초기 로딩 시 지도 API 호출 금지


### 5.3 성능 관련

* 불필요한 API 호출 금지
* 데이터는 클라이언트에서 필터링

***6. 디자인 관련 
 디자인은 'https://nininik.github.io/swim-finder/'를 참조해줘


## 7. ✅ 결론

이 프로젝트는 다음 구조를 따른다:

* 데이터: JSON 기반 (Data.json)
* 정보 : Google
* 지도: 검색 후 Naver Map API 실행

👉 단순하지만 확장 가능한 구조로 설계한다.
